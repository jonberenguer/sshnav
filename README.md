# sshnav

A terminal UI for managing SSHFS mounts and SSH ProxyJump tunnels — built with [Bubbletea](https://github.com/charmbracelet/bubbletea).

---

## Requirements

| Tool | Purpose |
|------|---------|
| Go ≥ 1.22 | build |
| `sshfs` | mounting remote filesystems |
| `fusermount` | unmounting (part of `fuse` / `fuse3`) |
| `ssh` | ProxyJump tunnels |

```
# Debian/Ubuntu
sudo apt install sshfs fuse3

# Arch
sudo pacman -S sshfs

# Fedora
sudo dnf install fuse-sshfs
```

## Build & Install

```bash
# Build only (binary in current directory)
bash build.sh

# Build + install to /usr/local/bin
INSTALL_DIR=/usr/local/bin bash build.sh

# Custom install path
INSTALL_DIR=~/.local/bin bash build.sh
```

> Builds run inside Docker (`golang:1.22-alpine` → `scratch`). No local Go toolchain required.

## Usage

```
sshnav [--profiles-only]        Launch the TUI
sshnav export-ssh-config        Convert profiles.yaml → ~/.ssh/config format (stdout)
sshnav import-ssh-config        Convert ~/.ssh/config → profiles.yaml format (stdout)
```

### Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--profiles-only` | false | Skip `~/.ssh/config` — only load app-managed profiles |

## Config files

| File | Purpose |
|------|---------|
| `~/.config/sshnav/profiles.yaml` | App-managed profiles (read/write) |
| `~/.ssh/config` | Imported as read-only profiles |

### profiles.yaml example

```yaml
- name: my-nas
  host: 192.168.1.50
  user: admin
  port: 22
  identity_file: ~/.ssh/id_ed25519
  remote_path: /mnt/data
  mount_point: /mnt/nas
  sshfs_opts: reconnect,ServerAliveInterval=15

- name: prod-bastion
  host: internal.prod.example.com
  user: deploy
  proxy_jump: bastion.example.com
  local_forwards:
    - "8080:localhost:80"
  remote_forwards:
    - "9090:localhost:9090"
```

All fields except `name` and `host` are optional.

## Screens & Keybindings

### Dashboard (default)

| Key | Action |
|-----|--------|
| `↑↓` | Navigate profiles |
| `Enter` | Open SSHFS panel (if mount configured) or Proxy panel |
| `m` | Open SSHFS panel for selected profile |
| `p` | Open Proxy panel for selected profile |
| `n` | New profile |
| `e` | Edit selected profile (app-managed only) |
| `l` | Profile list / manager |
| `/` | Filter profiles |
| `Ctrl+C` | Quit (closes all active tunnels) |

Live status dots (`●`/`○`) are polled every 5 seconds via `/proc/mounts`.

### Profile List

| Key | Action |
|-----|--------|
| `↑↓` | Navigate |
| `Enter` / `e` | Edit selected |
| `n` | New profile |
| `d` / `Delete` | Delete (app-managed only, confirms with y/N) |
| `Esc` | Back to Dashboard |

### Profile Edit / Create

| Key | Action |
|-----|--------|
| `Tab` / `↓` | Next field |
| `Shift+Tab` / `↑` | Previous field |
| `Ctrl+S` | Save |
| `Enter` | Advance field (save on last) |
| `Esc` | Cancel |

Fields are split into sections: **SSH**, **SSHFS**, and **Proxy**.

### SSHFS Panel

| Key | Action |
|-----|--------|
| `m` | Mount / Unmount |
| `e` | Edit profile |
| `Esc` | Back to Dashboard |

Mount state is read from `/proc/mounts` (looks for `fuse.sshfs` entries).

### Proxy Panel

| Key | Action |
|-----|--------|
| `t` | Open / Close tunnel |
| `e` | Edit profile |
| `Esc` | Back to Dashboard |

The tunnel runs `ssh -N -J <proxy_jump> <host>` with `ServerAliveInterval=30`. The process is owned by sshnav and killed on close or quit.

## Notes

- `~/.ssh/config` hosts are **read-only** — they appear in the dashboard and panels but cannot be edited or deleted from within sshnav. Edit them in your normal SSH config workflow.
- SSHFS mounts survive sshnav exiting (they are kernel FUSE mounts). SSH tunnels do **not** — they are subprocess-owned and are killed on quit.
- `sshfs_opts` is passed as comma-separated `-o key=value` pairs. Refer to `man sshfs` for available options.
